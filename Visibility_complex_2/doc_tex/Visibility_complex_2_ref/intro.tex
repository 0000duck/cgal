\subsection*{Global names of nested types}

The class \ccc{Visibility_complex_2<VC2GeomTraits,VC2Items>} exports
the types \ccc{Bitangent_2}, \ccc{Vertex}, \ccc{Edge}, \ccc{Face},
\ccc{Constraint_input}. The various bitangent types, as well as
\ccc{Constraint_input} export the type \ccc{Type}.

This is fine for most uses. However, one cannot define a polymorphic
template function over one of theses types and instanciate it without
qualification. To solve this problem, the nested namespace
\ccc{CGAL::Visibility_complex_2_details} contains global names for these
types. They are \ccc{Bitangent_2<Gt>}, \ccc{Vertex<A>}, \ccc{Edge<A>},
\ccc{Face<A>} (the template arguments are unspecified),
\ccc{Constraint_input} and \ccc{Bitangent_type}.
