\ccDefinition
  
The concept \ccRefName\ describes the requirements for the primitives stored in the AABB data structure. The concept encapsulates a type for the input objects and an identifier type by which those primitives are referred to. The concept \ccRefName\ also refines the concepts DefaultConstructible and Assignable.

\ccTypes

\ccNestedType{Object}{Type of input objects.}

\ccNestedType{Id}{Type of identifiers by which the input objects are referred to.}


% variable name
\ccCreationVariable{primitive}  

\ccOperations

\ccMethod{Object object();}
{Returns the geometric object wrapped inside the Primitive.}

\ccMethod{Id id();}
{Returns the corresponding identifier. This identifier is only used as a reference for the objects in the output of the \ccc{AABB_tree} methods.}

% \ccHasModels
% \ccc{AABB_primitive_base<ID>}

\ccSeeAlso
\ccc{AABB_tree<Traits>}

\ccExample

Usually, the \ccc{Primitive} type is a wrapper around a \ccc{Handle}. Assume for instance that the input objects are the triangle faces of a mesh stored as a \ccc{CGAL::Polyhedron}, the Primitive type would be a wrapper around the \ccc{Face_handle}  type of the polyhedron. The \ccc{Data} would be a \ccc{Triangle_3}, the \ccc{Id} would be a \ccc{Face_handle}. \ccc{object()} would return a \ccc{Triangle_3} either constructed on the fly from the id (stored as face handle) or stored internally. \ccc{id()} would return the \ccc{Face_handle}. 

\end{ccRefConcept}
