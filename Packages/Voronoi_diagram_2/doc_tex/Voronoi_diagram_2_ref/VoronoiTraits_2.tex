\begin{ccRefConcept}{VoronoiTraits_2}

\ccDefinition

The concept \ccc{VoronoiTraits_2} defines the combinatorial and
geometric predicate requirements for the
\ccc{Voronoi_diagram_2<DG,VT>} class. The
\ccc{VoronoiTraits_2} concept essentially defines the requirements on
the predicates that determine whether a feature of the triangulated
Delaunay graph has a degenerate dual feature in the Voronoi diagram
that is the dual of the triangulated Delaunay graph.

\ccRefines
\ccc{DefaultConstructible}, \ccc{CopyConstructible}, \ccc{Assignable}

\ccTypes
\ccTwo{VoronoiTraits_2::Face_degeneracy_tester+}{}
%
\ccNestedType{Point_2}{A type for a point.}
\ccGlue
\ccNestedType{Site_2}{A type for the sites of the Voronoi diagram.}
\ccGlue
\ccNestedType{Delaunay_graph}{A type for the triangulated Delaunay
  graph. The type \ccc{Delaunay_graph} must be a model of the
  \ccc{DelaunayGraph_2} concept.}
\ccGlue
\ccNestedType{Edge_degeneracy_tester}{A type for the predicate that
  tests whether a Voronoi edge is degenerate, i.e., it has zero
  length. It must be a model of the \ccc{EdgeDegeneracyTester} concept.}
\ccGlue
\ccNestedType{Face_degeneracy_tester}{A type for the predicate that
  tests whether a Voronoi face is degenerate, i.e., it has zero
  area. It must be a model of the \ccc{FaceDegeneracyTester} concept.}
\ccGlue
\ccNestedType{Has_inserter}{A tag for determining if the Voronoi
  traits class provides a functor for inserting sites in the Delaunay
  graph. This tag is equal to either \ccc{CGAL::Tag_true} (a site
  inserter functor is available) ir \ccc{CGAL::Tag_false} (a site
  inserter functor is not available).}
\ccGlue
\ccNestedType{Has_nearest_site_2}{A tag for determining if the Voronoi
  traits class provides a functor for performing nearest site queries.
  This tag is equal to either \ccc{CGAL::Tag_true} (a nearest site
  query functor is available) or \ccc{CGAL::Tag_false} (a nearest site
  query functor is not available).}
\ccGlue
\ccNestedType{Access_site_2}{A type for a functor that accesses the
  sites associated with a vertex. It must be a model of the
  \ccc{AccessSite_2} concept.}
\ccGlue
\ccNestedType{Construct_dual_point_2}{A type for a functor that
  constructs the dual point of a face in the Delaunay graph, i.e., the
  Voronoi vertex in the Voronoi diagram given to a face in the
  Delaunay graph. It must be a model of the \ccc{ConstructDualPoint_2}
  concept.}
\ccGlue
\ccNestedType{Nearest_site_2}{A type for a functor that performs
  nearest site queries. It must be a model of the concept
  \ccc{DelaunayGraphNearestSite_2}. This type is required only if
  \ccc{Has_nearest_site_2} is equal to \ccc{CGAL::Tag_true}.}
\ccGlue
\ccNestedType{Site_inserter}{A type for a functor that inserts sites
  in the Delaunay graph. It must be a model of the \ccc{SiteInserter}
  concept. This type is required only if the \ccc{Has_inserter} tag
  is equal to \ccc{CGAL::Tag_true}.}

\ccHeading{Miscellaneous}
The following methods are important when the Voronoi traits maintain a
state. This can happen if we have caching Voronoi traits, i.e., when
we have Voronoi traits that cache the results of the edge and face
degeneracy testers.

%

  \ccc{vt} and \ccc{other} are swapped. This method should be
  preferred to \ccStyle{vt=other} or \ccStyle{vt(other)} if
  \ccc{other} is deleted afterwards.}

\ccMethod{bool is_valid();}{Tests the validity of the Voronoi traits.}
