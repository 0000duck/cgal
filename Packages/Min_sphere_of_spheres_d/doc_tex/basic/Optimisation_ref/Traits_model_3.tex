\begin{ccRefClass}{Min_sphere_of_spheres_d_traits_3<K,FT,UseSqrt,Algorithm>}

% -----------------------------------------------------------------------------
\ccDefinition

The class
\ccc{Min_sphere_of_spheres_d_traits_3<K,FT,UseSqrt,Algorithm>} is a
model for concept \ccc{MinSphereOfSpheresTraits}.  It uses the types
\ccc{Point_3} and \ccc{Weighted_point} from {\cgal} to represent
spheres.

\ccIsModel
\ccc{MinSphereOfSpheresTraits}

\ccParameters
The last two template parameters, \ccc{UseSqrt} and \ccc{Algorithm},
have default arguments, namely \ccc{CGAL::Tag_false} and
\ccc{CGAL::Default_algorithm}, respectively.

 The template parameters of class
\ccc{Min_sphere_of_spheres_d_traits_3<K,FT,UseSqrt,Algorithm>} must
fulfill the following requirements:

\ccNestedType{K}{is a model for \ccc{Kernel}.}

\ccNestedType{FT}{is a number type, which fulfills the requirements of
type \ccc{FT} of concept \ccc{MinSphereOfSpheresTraits}: It must be
either \ccc{double} or an exact number type.}

\ccNestedType{UseSqrt}{fulfills the
requirements of type \ccc{Use_square_roots} of concept
\ccc{MinSphereOfSpheresTraits}:  It must be either \ccc{Tag_true} or \ccc{Tag_false}.}

\ccNestedType{Algorithm}{fulfills the requirements of type
\ccc{Algorithm} of concept \ccc{MinSphereOfSpheresTraits}: It must be either \ccc{Default_algorithm}, \ccc{LP_algorithm} or \ccc{Farthest_first_heuristic}.}

\ccTypes
In addition to the types required by the concept
\ccc{MinSphereOfSpheresTraits}, this model also defines the types
\ccc{Weight} and \ccc{Point}.  Here's the complete list of defined types:

\ccNestedType{FT}{}

\ccNestedType{Use_square_roots}{}

\ccNestedType{Algorithm}{}

\ccNestedType{Weight}{is a typedef to to the template parameter \ccc{FT}}

\ccNestedType{Point}{is a typedef to to \ccc{CGAL::Point_3<K>}.}

\ccNestedType{Sphere}{is a typedef to to
  \ccc{CGAL::Weighted_point<Point,Weight>}.}

\ccNestedType{Coordinate_iterator}{is a typedef to to
\ccc{Point::Cartesian_const_iterator}.}


\ccAccessFunctions

The class provides the access functions required by the model
\ccc{MinSphereOfSpheresTraits}; they simply map to the corresponding
routines of class \ccc{Weighted_point<Point,Weight>}:

\ccCreationVariable{traits}

\ccMemberFunction{ FT radius(const
Sphere& s);}{ maps to \ccc{s.weight()}.}

\ccMemberFunction{ Coordinate_iterator center_coordinates_begin(const
Sphere& s);}{ maps to \ccc{s.point().cartesian_begin()}.}

\ccMemberFunction{ Coordinate_iterator center_coordinates_end(const
Sphere& s);}{ maps to \ccc{s.point().cartesian_end()}.}

\ccMemberFunction{ int dimension(const Sphere& s);}{ maps to \ccc{s.dimension()}.}

\end{ccRefClass}
