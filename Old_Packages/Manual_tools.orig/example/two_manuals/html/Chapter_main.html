Example User Manual

3.11. 2001/08/20
Lutz Kettner and Susan Hert

As of release 3.11 of the manual tools, it is possible to create more than one manual in a single document that share a table of contents, bibliography and index. This allows for easy hyperlinking between the different manuals while keeping them logically separate. This example illustrates the creation of a user and reference manual from a set of files whose names are distinct and thus do not require the use of the -extended option for the HTML conversion.

Creation of the manual with is done via 

   latex manual.tex
The HTML version is created with 
   cc_manual_to_html -o html manual.tex
Notice that this simple conversion is possible with HTML because the two chapter commands appear in files with differnet names (main.tex in subdirectory Example and intro.tex in subdirectory Example_ref). If the chapter command for the reference pages were in the file Example_ref/main.tex, one would have to use the -extended option of cc_manual_to_html to avoid name collisions and the -ref_manual option to generate the two separate manuals. See the directory ../two_manuals_ext_html for an example using these options. The directory ../one_manual contains an example of the creation of a single manual containing both the user and reference manuals with the same directory structure as this one.
